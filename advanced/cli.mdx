# CLI Reference

Tamagui provides a powerful CLI for building, generating, and managing your Tamagui projects.

## Installation

```bash
npm install --save-dev @tamagui/cli
```

Or use without installing:

```bash
npx @tamagui/cli <command>
```

### generate

Builds your Tamagui configuration and outputs CSS and type information.

From `code/core/cli/src/cli.ts:36-64`:

```bash
tamagui generate
```

#### Flags

- `--help` - Show help
- `--debug` - Enable debug mode
- `--verbose` - Verbose debug output

1. Loads your Tamagui configuration
2. Generates CSS from themes and tokens
3. Creates `.tamagui/prompt.md` with LLM-friendly config documentation

#### Example


### build

Pre-builds Tamagui components for optimization.

From `code/core/cli/src/cli.ts:155-210`:

```bash
tamagui build [directory]
```

#### Arguments

- `[directory]` - Directory or file to build (default: current directory)

#### Flags

- `--help` - Show help
- `--debug` - Debug mode
- `--verbose` - Verbose output
- `--dry-run` - Preview changes without writing files
- `--target <platform>` - Target platform: `web`, `native`, or `both` (default: `both`)
- `--include <pattern>` - File pattern to include
- `--exclude <pattern>` - File pattern to exclude
- `--output <directory>` - Output directory for built files
- `--output-around` - Create `.native.tsx` files alongside source files
- `--expect-optimizations <n>` - Assert minimum number of optimizations

#### Run Command Syntax

From `code/core/cli/src/cli.ts:171-181`:

```bash
tamagui build [dir] -- <command>
```

Runs a command after optimization, then auto-restores files.

#### Examples

```bash
# Build current directory
tamagui build

# Build specific directory
tamagui build ./src/components

# Build for web only
tamagui build --target web

# Dry run (preview changes)
tamagui build --dry-run

# Include/exclude patterns
tamagui build --include "**/*.tsx" --exclude "**/*.test.tsx"

# Output to separate directory
tamagui build ./src --output ./dist

# Create .native.tsx files
tamagui build --output-around

# Expect minimum optimizations (useful in CI)
tamagui build --expect-optimizations 10

# Run tests with optimized code, then restore
tamagui build -- npm test
```

#### Build Output

From `code/core/cli/src/build.ts:17-23`:

```tsx
type BuildStats = {
```

### upgrade

Upgrades all Tamagui packages in your workspace.

From `code/core/cli/src/cli.ts:212-234`:

```bash
tamagui upgrade
```

Shorthand: `tamagui up`

#### Flags

- `--help` - Show help
- `--debug` - Debug mode
- `--from <version>` - Source version
- `--to <version>` - Target version
- `--changelog-only` - Only show changelog without upgrading
- `--dry-run` - Preview changes without upgrading

#### Examples

```bash
# Upgrade to latest
tamagui upgrade

# View changelog only
tamagui up --changelog-only

# Dry run
tamagui up --dry-run

# Upgrade to specific version
tamagui up --to 1.50.0
```

## Build System Details

### Platform-Specific Files

From `code/core/cli/src/build.ts:141-181`:

The build command intelligently handles platform-specific files:

```
// Base file
Button.tsx          → optimized for both platforms

// Platform-specific files
Button.web.tsx      → only optimized for web
Button.native.tsx   → only optimized for native
Button.ios.tsx      → only optimized for native
Button.android.tsx  → only optimized for native
```

**Rules:**
- Base file is optimized for platforms without specific files
- If both `.web.tsx` and `.native.tsx` exist, base file is not modified
- Platform extensions: `.web`, `.native`, `.ios`, `.android`

### CSS Import Injection

From `code/core/cli/src/build.ts:40-51`:

The build command automatically injects CSS imports:

```tsx
// Before
export const Button = styled(Stack, { ... })

// After
import "./_Button.css"
export const Button = styled(Stack, { ... })

// With 'use client'
'use client'
import "./_Button.css"
export const Button = styled(Stack, { ... })
```

### File Restoration

When using `--` to run commands:

1. Build optimizes files
2. Command runs with optimized code
3. Files are automatically restored if unchanged
4. Modified files during command are skipped (not restored)

## Common Workflows

### Development

```bash
# Check dependencies
tamagui check

# Generate types and CSS
tamagui generate
```

### Production Build

```bash
# Pre-optimize components
tamagui build ./src/components --target web

# Generate CSS for production
tamagui generate-css --output ./public/tamagui.css
```

### CI/CD

```bash
# Ensure minimum optimizations (fail if below threshold)
tamagui build --expect-optimizations 50

# Run tests with optimized code
tamagui build -- npm test
```

### Upgrading

```bash
# Check what will change
tamagui upgrade --dry-run --changelog-only

# Upgrade
tamagui upgrade
```

## Troubleshooting

### Build Issues

**"No tamagui config found"**
Ensure `tamagui.config.ts` exists and exports a valid config.

**"Expected X optimizations but got Y"**
Check that your components use Tamagui's `styled()` API.

### CSS Generation

**"Output CSS is empty"**
Ensure your config includes themes and tokens.

### Package Management

**"Version conflicts detected"**
Run `tamagui check` to see conflicts, then `tamagui upgrade` to fix.
