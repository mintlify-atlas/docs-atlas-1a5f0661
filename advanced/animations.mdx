# Advanced Animations

Tamagui's animation system uses a driver-based architecture that supports multiple animation backends. This guide covers advanced animation techniques and driver integration.

## Animation Drivers

Tamagui provides several animation drivers, each optimized for different platforms:

### React Native Driver

The React Native driver (`@tamagui/animations-react-native`) uses `Animated.Value` for native performance.

```tsx
import { createAnimations } from '@tamagui/animations-react-native'

export const animations = createAnimations({
  fast: {
    type: 'spring',
    damping: 20,
    mass: 1.2,
    stiffness: 250,
  },
  smooth: {
    type: 'spring',
    damping: 10,
    mass: 0.9,
    stiffness: 100,
  },
  slow: {
    type: 'timing',
    duration: 400,
  },
})
```

### CSS Driver

The CSS driver (`@tamagui/animations-css`) generates CSS transitions for web.

```tsx
import { createAnimations } from '@tamagui/animations-css'

export const animations = createAnimations({
  bouncy: 'ease-in 200ms',
  lazy: 'ease-in 600ms',
  quick: 'ease-in 100ms',
  slow: 'ease-in 500ms',
  tooltip: 'ease-in 400ms',
})
```

### Motion Driver (Recommended for Web)

The Motion driver (`@tamagui/animations-motion`) uses [motion](https://motion.dev) for high-performance web animations.

```tsx
import { createAnimations } from '@tamagui/animations-motion'

export const animations = createAnimations({
  quick: {
    type: 'spring',
    damping: 20,
    mass: 1.2,
    stiffness: 250,
  },
  bouncy: {
    type: 'spring',
    damping: 9,
    mass: 0.9,
    stiffness: 150,
  },
  lazy: {
    type: 'spring',
    damping: 18,
    stiffness: 50,
  },
})
```

## Enter/Exit Animations

Tamagui supports specialized enter and exit animations using the presence system.

### State-Based Animation Selection

From `code/core/animations-react-native/src/createAnimations.tsx:519-561`:

```tsx
function getAnimationConfig(
  key: string,
  animations: AnimationsConfig,
  transition?: TransitionProp,
  animationState: 'enter' | 'exit' | 'default' = 'default'
): AnimationConfig {
  const normalized = normalizeTransition(transition)
  const shortKey = transformShorthands[key]

  // Check for property-specific animation
  const propAnimation = normalized.properties[key] ?? normalized.properties[shortKey]

  let animationType: string | null = null
  let extraConf: any = {}

  if (typeof propAnimation === 'string') {
    animationType = propAnimation
  } else if (propAnimation && typeof propAnimation === 'object') {
    animationType = propAnimation.type || getEffectiveAnimation(normalized, animationState)
    extraConf = propAnimation
  } else {
    animationType = getEffectiveAnimation(normalized, animationState)
  }

  // Apply global delay
  if (normalized.delay && !extraConf.delay) {
    extraConf = { ...extraConf, delay: normalized.delay }
  }

  const found = animationType ? animations[animationType] : {}
  return {
    ...found,
    ...normalized.config,
    ...extraConf,
  }
}
```

### Using AnimatePresence

```tsx
import { AnimatePresence } from '@tamagui/animate-presence'

function MyComponent() {
  const [show, setShow] = useState(false)

  return (
    <AnimatePresence>
      {show && (
        <YStack
          animation="quick"
          enterStyle={{ opacity: 0, scale: 0.9 }}
          exitStyle={{ opacity: 0, scale: 0.95 }}
        >
          Content
        </YStack>
      )}
    </AnimatePresence>
  )
}
```

## Dynamic Theme Values

Tamagui resolves dynamic theme values during animation:

From `code/core/animations-react-native/src/createAnimations.tsx:16-23`:

```tsx
const resolveDynamicValue = (value: any, isDark: boolean): any => {
  if (value && typeof value === 'object' && 'dynamic' in value) {
    const dynamicValue = isDark ? value.dynamic.dark : value.dynamic.light
    return dynamicValue
  }
  return value
}
```

This enables theme-aware animations:

```tsx
<YStack
  animation="quick"
  backgroundColor={isDark ? '$background' : '$backgroundLight'}
/>
```

## Custom Animation Hooks

### useAnimatedNumber

Create custom animated values:

```tsx
import { useAnimatedNumber } from '@tamagui/core'

function MyComponent() {
  const rotation = useAnimatedNumber(0)

  return (
    <YStack
      onPress={() => {
        rotation.setValue(360, { type: 'spring', damping: 20 })
      }}
    />
  )
}
```

### useAnimatedNumberReaction

React to animated value changes:

```tsx
import { useAnimatedNumberReaction } from '@tamagui/core'

function MyComponent() {
  const value = useAnimatedNumber(0)

  useAnimatedNumberReaction({ value }, (current) => {
    console.log('Current value:', current)
  })
}
```

## Best Practices

1. **Use native-compatible properties** (`transform`, `opacity`) for best performance
2. **Avoid animating layout properties** like `width`, `height` when possible
3. **Use spring animations** for natural, physical motion
4. **Batch animations** by animating multiple properties with one `animation` prop
5. **Test on device** - animations may perform differently on real hardware
6. **Use AnimatePresence** for enter/exit animations
7. **Prefer the Motion driver** for web applications
