---
title: 'Metro Bundler Integration'
description: 'Configure Tamagui with Metro for React Native and Expo projects'
---

<Info>
  **Prerequisites**:
  - React Native 0.70 or higher
  - Node.js 18 or higher
  - Metro bundler configured
</Info>

The Tamagui Metro plugin enables CSS generation and configuration loading for React Native projects.

## Installation

<Steps>
<Step title="Install the Metro plugin">

<CodeGroup>
```bash npm
npm install @tamagui/metro-plugin
```

```bash yarn
yarn add @tamagui/metro-plugin
```

```bash pnpm
pnpm add @tamagui/metro-plugin
```
</CodeGroup>

</Step>

<Step title="Configure Metro">

Update your `metro.config.js`:

```js metro.config.js
const { getDefaultConfig } = require('expo/metro-config')
const { withTamagui } = require('@tamagui/metro-plugin')

// Get base Metro config
const config = getDefaultConfig(__dirname, {
  isCSSEnabled: true,
})

// Add Tamagui plugin
module.exports = withTamagui(config, {
  components: ['tamagui'],
  config: './tamagui.config.ts',
})
```

<Note>
For plain React Native (non-Expo), use `@react-native/metro-config` instead of `expo/metro-config`.
</Note>

</Step>

<Step title="Create Tamagui config">

Create `tamagui.config.ts`:

```ts tamagui.config.ts
import { config } from '@tamagui/config/v3'
import { createTamagui } from 'tamagui'

const tamaguiConfig = createTamagui(config)

export default tamaguiConfig

export type Conf = typeof tamaguiConfig

declare module 'tamagui' {
  interface TamaguiCustomConfig extends Conf {}
}
```

</Step>

<Step title="Create build configuration">

Create `tamagui.build.ts` for CSS generation:

```ts tamagui.build.ts
import type { TamaguiBuildOptions } from 'tamagui'

export default {
  components: ['tamagui'],
  config: './tamagui.config.ts',
  outputCSS: './tamagui.generated.css',
} satisfies TamaguiBuildOptions
```

</Step>

<Step title="Generate CSS">

Run the Tamagui CLI to generate CSS:

```bash
npx tamagui generate
```

This creates `tamagui.generated.css` based on your config.

</Step>

<Step title="Import CSS in your app">

Import the generated CSS in your root component:

```tsx App.tsx
import './tamagui.generated.css'
import { TamaguiProvider } from 'tamagui'
import config from './tamagui.config'

export default function App() {
  return (
    <TamaguiProvider config={config}>
      {/* Your app content */}
    </TamaguiProvider>
  )
}
```

</Step>
</Steps>

## Configuration Options

The `withTamagui` function accepts these options:

```ts
type MetroTamaguiOptions = {
  // Path to your Tamagui config file
  config?: string
  
  // Component packages to process
  components?: string[]
  
  // Platform target ('native' for React Native)
  platform?: 'native' | 'web'
  
  // Disable style extraction
  disableExtraction?: boolean
  
  // @deprecated - use tamagui generate instead
  cssInterop?: boolean
}
```

## How It Works

The Metro plugin is simplified in recent versions:

1. **Loads your config**: Reads `tamagui.config.ts` and `tamagui.build.ts`
2. **Enables CSS**: Adds `.css` to Metro's `sourceExts`
3. **Delegates extraction**: Style extraction is handled by `tamagui generate` CLI
4. **Stores options**: Makes config available to other tools

```js
// What the plugin does internally:
metroConfig.resolver = {
  ...metroConfig.resolver,
  sourceExts: [...metroConfig.resolver.sourceExts, 'css'],
}

metroConfig.transformer = {
  ...metroConfig.transformer,
  tamagui: loadedOptions,
}
```

## Plain React Native

For React Native without Expo:

```js metro.config.js
const { getDefaultConfig, mergeConfig } = require('@react-native/metro-config')
const { withTamagui } = require('@tamagui/metro-plugin')

const config = getDefaultConfig(__dirname)

// Enable CSS support
const cssConfig = {
  resolver: {
    sourceExts: [...config.resolver.sourceExts, 'css'],
  },
}

module.exports = withTamagui(
  mergeConfig(config, cssConfig),
  {
    components: ['tamagui'],
    config: './tamagui.config.ts',
  }
)
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Warning: cssInterop is deprecated">
    Remove the `cssInterop` option from your config:
    
    ```js
    // Before
    withTamagui(config, {
      cssInterop: true, // Remove this
    })
    
    // After
    withTamagui(config, {
      components: ['tamagui'],
      config: './tamagui.config.ts',
    })
    ```
    
    Use `tamagui generate` instead for CSS generation.
  </Accordion>

  <Accordion title="Error: Cannot resolve .css files">
    Ensure CSS is enabled in Metro:
    
    ```js
    const config = getDefaultConfig(__dirname, {
      isCSSEnabled: true, // For Expo
    })
    
    // Or manually add to sourceExts:
    config.resolver.sourceExts = [
      ...config.resolver.sourceExts,
      'css',
    ]
    ```
  </Accordion>

  <Accordion title="Styles not applying">
    Verify you've completed all steps:
    
    1. Created `tamagui.build.ts`
    2. Ran `npx tamagui generate`
    3. Imported the generated CSS:
    
    ```tsx
    import './tamagui.generated.css'
    ```
  </Accordion>

  <Accordion title="Metro bundler cache issues">
    Clear Metro cache and restart:
    
    ```bash
    # Expo
    npx expo start --clear
    
    # React Native
    npx react-native start --reset-cache
    ```
  </Accordion>

  <Accordion title="Config not loading">
    Check the path to your config file is correct:
    
    ```js
    withTamagui(config, {
      config: './tamagui.config.ts', // Relative to project root
    })
    ```
    
    The plugin uses `loadTamaguiBuildConfigSync` from `@tamagui/static`.
  </Accordion>
</AccordionGroup>

## Advanced Usage

### Custom Metro Transformer

If you have a custom Metro transformer, ensure it processes Tamagui files:

```js metro.config.js
const { withTamagui } = require('@tamagui/metro-plugin')

const config = {
  transformer: {
    getTransformOptions: async () => ({
      transform: {
        experimentalImportSupport: false,
        inlineRequires: true,
      },
    }),
  },
}

module.exports = withTamagui(config, {
  components: ['tamagui'],
  config: './tamagui.config.ts',
})
```

### Platform-Specific Configuration

Set platform in the options:

```js metro.config.js
const isWeb = process.env.TAMAGUI_TARGET === 'web'

module.exports = withTamagui(config, {
  components: ['tamagui'],
  config: './tamagui.config.ts',
  platform: isWeb ? 'web' : 'native',
})
```

### Monorepo Setup

For monorepos, use `watchFolders`:

```js metro.config.js
const path = require('path')
const { withTamagui } = require('@tamagui/metro-plugin')

const projectRoot = __dirname
const workspaceRoot = path.resolve(projectRoot, '../..')

const config = {
  watchFolders: [workspaceRoot],
  resolver: {
    nodeModulesPaths: [
      path.resolve(projectRoot, 'node_modules'),
      path.resolve(workspaceRoot, 'node_modules'),
    ],
  },
}

module.exports = withTamagui(config, {
  components: ['tamagui'],
  config: './tamagui.config.ts',
})
```

```js
// Remove this option
cssInterop: true
```

</Step>

<Step title="Create tamagui.build.ts">

```ts tamagui.build.ts
export default {
  components: ['tamagui'],
  config: './tamagui.config.ts',
  outputCSS: './tamagui.generated.css',
}
```

</Step>

<Step title="Run tamagui generate">

```bash
npx tamagui generate
```

</Step>

<Step title="Import CSS">

```tsx
import './tamagui.generated.css'
```

</Step>
</Steps>
